## 이 스킬이 하는 일
프로젝트의 모든 도메인 스킬을 순서대로 불러와 완전한 기능 구현을 오케스트레이션합니다:
1. 기능 요구사항을 이해하고 코드베이스에 매핑
2. React 컴포넌트 빌드/수정 (`react-component` 패턴 사용)
3. 게임 데이터/스테이지 수정 (`game-data` 패턴 사용)
4. 터미널 스타일링 적용 (`terminal-ui` 패턴 사용)
5. 빌드 통과 및 기능 동작 확인

## 언제 사용하나요?
아래와 같은 작업에서 이 스킬을 사용하세요:
- 새 게임 메커닉 추가 (인벤토리, 스킬 체크, 세이브 슬롯 등)
- 새 UI 패널/오버레이 추가 (설정, 도감, 업적 등)
- 완전한 스토리 분기 구현 (새 스테이지 + UI 변경)
- 컴포넌트, 데이터, 스타일 중 2개 이상을 건드리는 모든 작업

## 워크플로우
1. **기획:** `react-component`, `game-data`, `terminal-ui` 스킬을 모두 로드합니다. 관련 소스 파일을 읽고 구현 계획을 수립합니다.
2. **데이터:** 새 게임 로직이 필요하면 `src/data/scenario.js` 또는 `src/hooks/useGameState.js`를 먼저 수정합니다.
3. **컴포넌트:** `src/components/<카테고리>/` 폴더에 JSX 컴포넌트를 생성/수정합니다.
4. **연결:** 새 컴포넌트를 `src/App.jsx`나 부모 컴포넌트에 import하여 통합합니다.
5. **스타일:** Tailwind 클래스로 터미널 미학 적용 (색상, 애니메이션, 반응형 레이아웃).
6. **검증:** `npm run dev`를 실행해 기능이 정상 동작하는지 확인합니다. 콘솔 에러도 체크합니다.
7. **마무리:** 전환 효과, 로딩 상태, 엣지 케이스 처리가 부드러운지 다듬습니다.

## 시작 전 확인
다음 상황에서는 사용자에게 먼저 확인 질문을 하세요:
- 기능 범위가 모호할 때
- 여러 가지 접근 방법이 있을 때
- 핵심 데이터 스키마 변경이 필요할 때
